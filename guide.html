Installation Steps

Standard Installation (Admin)
1. Run installer as Administrator
patriot_setup.exe

2. Verify installation
sc query PatriotUltraBlocker
Installer verifies SHA-256 for every binary before extraction. Do not modify or repackage the installer.

Howto Use

First Launch

- Patriot starts minimized to the system tray.
- Double-click the shield icon in the tray to open the dashboard.
- On first launch, the threat database is downloaded automatically (requires internet; ~2 MB download).
- Dashboard appears after database is ready.

SIEM Integration (Wazuh)

Plaintext connections to remote hosts are blocked by default. Always use TLS via stunnel.

Stunnel Configuration

stunnel.conf
[wazuh-tls]
client  = yes
accept  = 127.0.0.1:51514
connect = WAZUH_SERVER_IP:1514
CAfile  = /etc/stunnel/wazuh-ca.crt
verify  = 2

Configuration

config.ini
[siem]
siem_active=true
siem_host=WAZUH_SERVER_IP
siem_port=1514
siem_use_tls_proxy=true
siem_tls_local_port=51514

Cloud Update

IOC databases are updated from the cloud every 6 hours asynchronously. Cloud URLs are stored in an encrypted file using DPAPI.

DatabaseLocal FileContent
Applicationsapplication_security.csvHash + name + threat application category
Ransomwareransomware_hash.csvHash + family + severity level
Network Denylistnetwork_denylist.csvIP/domain + threat category
Each file is SHA-256 verified via a sidecar file before use (fail-closed). If the sidecar is unavailable or the hashes don't match, the old file is retained.

Quarantine

Malicious files are quarantined in .qtn — AES-256 encrypted with a key based on ComputerName + UserName + install date. The files are stored in %ALLUSERSPROFILE%\Patriot\quarantine\.

Restore and whitelist+restore operations require administrator password verification. Permanent delete operations require explicit confirmation.

AutoRollback

Document snapshots are taken every 2–30 minutes (configurable via backup_interval_min). When ransomware is detected and blocked, documents modified during the process are automatically restored from the last snapshot.

The backup quota is controlled via backup_quota_mb (default 2048 MB). When the quota is full, older snapshots are automatically deleted (FIFO).
